About the Interdisciplinary Earth Data Alliance (IEDA) Integrated Catalog

This application searches a catalog of resources under the stewardship of partners in the Interdisciplinary Earth Data Alliance (IEDA). Catalog records are harvested from the EarthChem Library, US Antarctic Program Data Center, Marine-Geo Digital Library and Academic Seismic Portal partners of the Interdisciplinary Earth Data Alliance. Funding is provided by the National Science Foundation (Award #1636653).

Search options:

Enter any text in the Search box (upper left); the content searched includes title, author, abstract, and keywords. Click on the green magnifying glass button to execute the search. Applied search terms will appear in the list beneath the 'Filters' heading at the top center. Each applied search term has an 'X' next to it. Clicking on the 'X' will remove that filter criteria and update the facet counts.

The number of items matching the current filter is displayed in the upper right. The results panel displays 10 result cards at a time. Advance or move back through the current selected set using the arrows next to the page number label in the upper right.

The result set can be ordered by relevance (as determined by the search engine), by title, or by reported acquisition date (which is likely to be different from the release or publication date displayed in the result cards on the web page).

Click on one of the titles beneath 'Search facets' (left side) to see a list of facet terms. If a search term has already been applied, the facet list represents results for the currently selected set. Clicking on a term in the facet list will add that as a filter criteria and update facet counts.

Facets:

  • 'Investigator/Author' names are those provided by the data submitters, with review by IEDA curators.
  • 'Acquisition date interval' provides a plot with dots indicating the begin and end dates of acquisition intervals as reported by the submitter; the relationship of this reported interval to the data is thus variable. Use with caution, because the begin and end interval dots are not differentiated. Apply the filter by clicking on the small date range text ('yyyy..yyyy') to the left of the interval selection box in the lower right of the facet card. Note that in order to change the criteria, the 'Acquisition date interval' filter should be removed and reapplied.
  • 'Publication or release date' provides a plot with dots indicating when the dataset was published or released, or will be released if it is under moratorium. See notes on the 'Acquisition date interval' facet (above) for usage instructions.
  • 'IEDA Topic', 'IEDA Feature of Interest', and 'IEDA Data Type Categories' are keywords from a controlled vocabulary reviewed by IEDA curators. The lists provide a good overview of the kinds of content in the catalog and the frequency of different content types.
  • 'Instrument and platform' facets are applied systematically for datasets in the Marine-Geo Digital Library, and are provided by submitters and therefore less consistent for other partner systems. It cannot be assumed that all datasets derived with a particular instrument or platform have been associated with these facets.
  • 'Tags' are uncontrolled index terms provided by data submitters or IEDA curators to assist with data discovery. Unlike the other facet lists, this list is sorted according to the frequency (most common first) that a term is provided as a tag within the current selected records.
  • 'IEDA partner system' allows filtering for all catalog items from a particular data repository.
  • 'Projects, cruises, programs' are applied systematically for datasets in the Marine-Geo Digital Library, and are provided by submitters and therefore less consistent, for other partner systems. It cannot be assumed that all datasets related to a particular initiative have been associated with these facets.
  • 'Place' keywords are provided by data submitters, and are not vetted by IEDA curators. It cannot be assumed that all datasets related to a particular place have been associated with these facets.

  • 'Map' provides for geospatial filtering. Pan and zoom to an area of interest, and click the 'Intersects' radio button to find all datasets with bounding boxes intersecting the current view extent. Clicking 'Within' will find all resources with bounding boxes completely within the current view extent. Note that the spatial search function implemented by the indexing system does not function as expected for boxes intersecting the antimeridian (180E). For data related to ship tracks or widely scattered individual samples, the bounding box extent will likely misrepresent the actual data coverage. Not all metadata records include a geographic bounding box, and these will not be found using the Map facet filter. Try using place name search in the Search box if the map search does not find expected results.

    Dots on the map represent the centroid of provided bounding boxes, aggregated into latitude-longitude grid cells that are appropriate to the current map extent. Larger dots indicate more hits within the grid cell. Zooming in on a dot will reduce the grid cell size, so smaller dots will appear more accurately representing individual centroid locations. When the map facet is open, mousing-over a search result in the results panel will display a red rectangle in the map showing the bounding box if a bounding box was provided in the metadata for the resource.

The 'Sign In' link in the upper right is for administrator use only.

For each search result displayed in the center part of the window, the resource title, authors or originators, release or publication date, and hosting partner system are reported at the top, followed by the abstract provided in the metadata. The links at the bottom of the card provide access to a web page presenting all metadata provided for the resource, the ISO 19139 XML record that was harvested to register the resource in the catalog, and a JSON document that is the actual document indexed for the catalog search. If links to the resource are provided in the metadata, these will be in the 'Links to resource' drop down on the right end of the links list. One or more links might be offered.

Technical Information

The catalog search is implemented using the ESRI open-source Geoportal Software, v. 2.5, customized in an IEDA fork of the ESRI gitHub repository. Metadata are harvested into the catalog as ISO 19139 xml files from web accessible folders, and metadata are available as a web page, ISO 19139 XML, or JSON via links with each search result.

The catalog can be searched via API using the OGC Catalog Service for the Web (CSW), version 3.0.0 or OpenSearch. Capabilities are described in the CSW getCapabilities document and the OpenSearch description

The APIs return metadata in CSW records or Atom XML format, both of which contain basic Dublin Core metadata elements.
